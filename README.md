# MKT-AD-archive

카카오톡 비즈보드 등 광고 소재 스크린샷을 브랜드·매체·소재 사이즈/형식·날짜별로 모아 정리하고 검색하는 팀 공유 웹앱입니다.

## 접속 방법

**https://seoyounglee0628.github.io/MKT-AD-archive/** 를 팀원들과 북마크로 공유하세요. 서버를 켜거나 터미널을 쓸 필요 없고, 누가 업로드하든 같은 데이터를 모두가 봅니다.

(로컬에서 개발/테스트하고 싶으면 `./start.sh`로 로컬 서버를 띄워 http://localhost:8787 로 접속할 수도 있습니다. 로컬이든 배포된 사이트든 같은 Supabase 백엔드를 보기 때문에 데이터는 항상 동일합니다.)

## 데이터 수집 방법 (중요)

이 프로그램은 광고를 자동으로 수집하지 않습니다. 카카오는 비즈보드 광고 아카이브 공개 API를 제공하지 않고, 자동으로 여러 계정을 돌려 타깃별 노출 광고를 긁는 방식은 카카오 이용약관 위반 소지가 있습니다.

대신 이렇게 사용하세요:
1. 실제 기기(본인 폰, 테스트 기기 등)에서 노출된 광고를 스크린샷으로 캡처
2. 이 앱에서 "+ 광고 추가"로 업로드 → 필요하면 잘라내기(크롭)로 불필요한 여백 제거
3. 브랜드/카피 문구/소재 사이즈/소재 형식은 자동 인식되어 채워짐 (아래 참고), 확인 후 필요하면 수정
4. 매체(Google·Meta·Kakao·CPA·TMS·LMS·앱푸시·브랜드메시지·알림톡 중 선택)/캡처 날짜 등 나머지 항목을 태그
5. 갤러리에서 검색·필터로 모아보기 — 다른 팀원이 올린 것도 "🔄 새로고침"을 누르면 바로 보입니다
6. 상세보기에서 "⬇️ 다운로드"로 원본 이미지를 내 컴퓨터에 저장할 수 있습니다

## 이미지 자르기 (크롭)

이미지를 업로드하면 저장 전에 크롭 화면이 먼저 뜹니다. 원하는 영역으로 자른 뒤 "자르기 적용"을 누르면 그 결과가 저장되고, 자를 필요가 없으면 "원본 그대로 사용"을 누르면 됩니다. 저장 후에도 미리보기 아래 "✂️ 다시 자르기" 버튼으로 다시 자를 수 있습니다. 자르기 이후에 카피/브랜드 자동 인식이 실행되므로, 브랜드명이 있는 부분을 잘라내면 자동 인식이 안 될 수 있습니다.

## 자동 인식 (OCR — 브랜드 / 카피 문구)

이미지를 업로드하면 브라우저 안에서 Tesseract.js로 텍스트를 인식해서 카피 문구를 채우고, 인식된 텍스트에 알려진 브랜드명이 있으면 브랜드도 자동으로 채웁니다. 인식 전에 이미지를 확대·흑백 대비 강조 처리하고, 배너 광고처럼 텍스트가 흩어진 레이아웃에 맞는 인식 모드를 사용해서 정확도를 높였습니다.

로고만 있고 텍스트가 없거나, 목록에 없는 브랜드는 자동으로 인식되지 않을 수 있습니다 — 자동 채움 결과는 항상 확인 후 필요하면 수정해주세요. `app.js`의 `KNOWN_BRANDS`에 브랜드를 추가하면 인식 가능한 브랜드를 늘릴 수 있습니다. 카카오톡 배너 특유의 작은 폰트·화려한 배색은 로컬 OCR의 한계로 여전히 정확도가 떨어질 수 있는데, 유료 OCR API(Naver Clova OCR, Google Vision 등) 연동으로 개선할 수도 있습니다 — 필요하면 요청해주세요.

## 자동 인식 (소재 사이즈 / 소재 형식)

이미지를 업로드하면 가로/세로 비율을 계산해서 소재 사이즈(가로형·세로형·정방형)를 자동으로 채웁니다. 소재 형식은 업로드한 파일이 GIF면 "영상형"으로, 그 외에는 이미지의 배경색이 얼마나 단조로운지를 계산해서 배경이 단순하면 "오브젝트형"(아이콘/오브젝트가 얹힌 형태), 색이 다양하면 "이미지형"으로 추정합니다. 두 값 모두 어디까지나 대략적인 추정이라 항상 확인 후 필요하면 수정해주세요. (실제 동영상 파일 업로드는 지원하지 않습니다 — 캡처 특성상 이미지 스크린샷만 다룹니다.)

매체는 정해진 목록 중에서 고르는 드롭다운입니다. 목록을 더 추가하고 싶으면 `index.html`에서 `id="inMedia"`와 `id="fMedia"` 두 군데의 `<option>`을 같이 추가하면 됩니다.

## 데이터 저장 위치 (중요 — 로컬 전용에서 변경됨)

이전에는 브라우저 안에만 저장되는 완전 로컬 방식이었지만, **팀원끼리 데이터를 공유하기 위해 Supabase(공유 클라우드 DB + Storage)로 옮겼습니다.** 이제 광고 메타데이터는 Supabase Postgres 테이블(`ads`)에, 이미지는 Supabase Storage 버킷(`ad-images`)에 저장되고, 이 앱을 여는 모든 사람이 같은 데이터를 봅니다.

**보안 관련 안내**: 로그인 기능이 없어서, 이 앱 URL과 약간의 기술 지식이 있으면 누구나 데이터를 읽고 쓰고 지울 수 있는 완전 오픈 구조입니다(Row Level Security를 "전체 허용"으로 설정). 팀 내부에서만 링크를 공유하고, URL이 외부에 노출되지 않도록 주의해주세요. 더 강한 보안이 필요하면 간단한 로그인(공유 비밀번호 등)을 추가하는 것도 가능합니다.

무료 저장 용량은 1GB(전체 이미지 합산 기준)까지이고, 스크린샷 특성상 대략 1,000~3,000장 정도는 여유 있게 담을 수 있습니다. Supabase 프로젝트가 7일 동안 아무도 안 쓰면 자동으로 일시정지되니, 그럴 땐 [Supabase 대시보드](https://supabase.com/dashboard)에서 프로젝트를 열어 깨워주면 됩니다.

내보내기/가져오기 기능은 제거했습니다 (공유 DB로 옮기면서 백업 필요성이 줄어듦). 개별 이미지가 필요하면 상세보기의 "⬇️ 다운로드"를 사용하세요.

## 파일 구성

- `index.html` — 화면 구조
- `style.css` — 스타일
- `app.js` — 저장/필터/검색/크롭/OCR 자동 인식 로직 (Supabase + Cropper.js + Tesseract.js)
- `start.sh` — 로컬 개발 서버 실행 스크립트
